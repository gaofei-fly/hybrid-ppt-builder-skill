# PPT Prompt Templates

Use these Chinese templates when the user wants reusable prompts or a staged PPT-generation workflow.

## 总控 Prompt

```text
你是我的 PPT 制作助手。请严格按“需求确认 → 内容提炼 → 版式方案 → 分层制作 → 预览核对 → 交付”的流程执行。

我的默认策略是：
1. 背景相对复杂且通常不改，优先使用 PNG/JPEG 保持视觉一致。
2. 标题、正文、页面重点、标签、卡片、流程线、时间轴等需要调整的前景内容，使用 PowerPoint 原生文本框和形状。
3. 图标、复杂符号、装饰性复杂图形等通常不编辑的前景模块，可以用 SVG 一次性绘制。
4. 如果 Office 版本能稳定把 SVG 转换为 PowerPoint 形状，可以优先考虑“可转换 SVG”，但必须先验证；标题、正文和关键布局仍优先保留为原生对象。
5. 不要用整页 PNG 或单个 SVG 图片层冒充可编辑 PPT。

交付前必须导出预览图，检查页数、尺寸、背景、文字、对象数量和可编辑性。
```

## 制作模式选择 Prompt

```text
请先按对象类型选择制作方案：
- 背景、照片、复杂纹理：PNG/JPEG。
- 标题、正文、重点句、页码、标签：PowerPoint 原生文本。
- 卡片、网格、箭头、流程线、时间轴：PowerPoint 原生形状。
- 图标、复杂符号、装饰插画：SVG；如果可转换为形状，先测试再使用。
- 需要改数据的图表：PowerPoint 原生图表或表格。
- 不需要改的复杂示意图：SVG 或图片。

请说明每页每类对象的选择理由。
```

## 生成 Prompt

```text
请开始生成 PPTX。

要求：
- 使用 16:9 页面尺寸。
- 背景作为独立底图保留，不要重绘或变形。
- 文字和关键布局使用 PowerPoint 原生文本框和形状。
- 复杂但不常编辑的图标/符号可以作为独立 SVG 模块插入。
- 不要把整页前景合成一张图片。
- 中文字体优先使用 Microsoft YaHei。
- 生成后保存到指定路径。

同时输出预览 PNG 和预览联系表。
```

## 交付前审查 Prompt

```text
交付前请自查，不通过就继续修改：
1. PPT 能被 PowerPoint 打开。
2. 页数正确。
3. 页面尺寸正确。
4. 背景存在且未变形。
5. 前景不是整页图片。
6. 如果要求可编辑，每页应有多个 PowerPoint 原生对象，而不是只有背景图 + 一个前景图。
7. 如果使用 SVG，说明它是可编辑形状、可转换 SVG，还是仅作为稳定图形模块。
8. 导出预览图，检查文字重叠、溢出、过小、过淡、边框或阴影过重。
9. 检查中文是否乱码。
10. 报告文件路径、页数、可编辑性验证方式和仍需人工确认的点。
```

## 最常用一句话

```text
请把这个 PPT 的第 [页码范围] 页重新制作：背景保持不变，标题、文字、重点内容和主要排版做成 PowerPoint 可编辑的原生文本和形状；复杂但通常不编辑的图标或符号可用独立 SVG；生成新的 PPTX，并导出预览图检查。不要用整页图片或单个 SVG 图片层冒充可编辑内容。
```
