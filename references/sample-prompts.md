# Sample Prompts

Use these Chinese prompts when the user asks for reusable wording, staged workflows, or a copy/paste task recipe.

## 总控 Prompt

```text
你是我的正式报告型 PPT 制作助手。请按“输入角色识别 -> 背景/前景契约 -> 对象级制作方案 -> 分层生成 -> 预览验证 -> 交付报告”的流程执行。

默认交付是可编辑 PPTX：背景可以作为稳定 PNG/JPEG 底图保留，标题、正文、页码、标签、卡片、流程线、时间轴、图表标签和关键结论必须用 PowerPoint 原生文本/形状/表格/图表实现。复杂但通常不编辑的图标、技术符号和装饰线稿可以使用独立 SVG。

不要用整页 PNG、单个 SVG 图片层、截图式表格或不可编辑前景冒充可编辑 PPT。交付前必须导出预览图，并检查页数、16:9 尺寸、对象数量、媒体数量、原生文本、中文显示和可编辑性。
```

## 现有 PPT 重制

```text
请重制这个 PPT 的第 [页码范围] 页。

要求：
- 保留原背景或重建为稳定背景层，不要把背景上的装饰误当成可编辑正文。
- 标题、正文、重点句、页码、标签、卡片、箭头、流程线、时间轴和主要版式必须做成 PowerPoint 原生对象。
- 复杂但不常编辑的图标/符号可以作为独立 SVG 或图片模块保留。
- 不要把整页前景合成一张图片。
- 输出新的 PPTX，同时导出预览 PNG 和联系表。
- 交付时报告每页对象数量、媒体数量、原生文本检查结果和仍需人工确认的点。
```

## 稿件转正式汇报

```text
请把以下稿件整理成正式报告型 PPT。

要求：
- 保留原文的论点、术语、证据和结论，不要编造新事实。
- 把长段落压缩为页面标题、短 bullet、流程图、时间线、对比表或结论卡片。
- 支撑性解释放到备注区，页面正文保持可读。
- 使用统一的浅色正式报告背景系统，前景全部采用可编辑的 PowerPoint 原生对象。
- 如果需要新视觉系统，请先给出 2-3 页视觉样张供确认；如果我明确要求一次性生成，则跳过确认但仍需预览验证。
```

## 参考背景图判定

```text
请先判断这张图适合作为 PPT 母版背景，还是只适合作为风格参考。

请检查：
- 是否为 16:9 或可安全裁切到 16:9；
- 是否有干净的标题区、正文区、图表区和页脚区；
- 是否包含不应进入背景的页面主张、正文、标签、数据或结论；
- 背景图形是否会干扰小字号文字、图表或密集表格；
- 可以抽取哪些颜色、线宽、纹理、页眉页脚、技术视觉元素和安全内容区。

判断后给出背景/前景契约和下一步制作方案。
```

## Gate 1 背景候选确认

```text
这是正式报告 PPT 的背景候选。请先只评估背景，不制作完整 PPT。

请判断：
- 是否符合学术/工业正式报告气质；
- 主阅读区是否干净；
- 页眉、页脚、页码或分节标记是否稳定；
- 技术/行业视觉线索是否足够但不喧宾夺主；
- 是否需要改颜色、密度、线条、留白或安全区。

确认后再进入 2-3 页视觉样张。
```

## Gate 2 视觉样张确认

```text
请先生成 2-3 页视觉样张，不要直接生成完整 PPT：
- 封面或标题页；
- 一页密集正文/表格页；
- 一页流程、卡片或图解页。

请用样张验证背景和前景是否匹配、标题/正文层级是否清楚、卡片/线条/分割线/图表强调色是否合适、信息密度是否可读，以及整体是否像正式的学术/工业报告。
```

## SVG 可编辑性确认

```text
如果使用 SVG，请明确区分：
- SVG 源文件；
- PPT 中插入的 SVG 图片层；
- 可被 PowerPoint 转换的 SVG；
- PowerPoint 原生形状。

标题、正文、关键标签、数据、结论和主要排版不能只存在于不可编辑 SVG 或图片里。若声称 SVG 可编辑，请先验证转换后对象可选择、文字可编辑、分组可用且外观不漂移。
```

## 交付前审查

```text
交付前请自查，不通过就继续修改：
1. PPTX 能被 PowerPoint 打开，或说明使用了 LibreOffice/包检查 fallback。
2. 页数正确，页面为 16:9。
3. 背景存在、未变形，且不承载必须编辑的页面逻辑。
4. 前景不是整页图片；可编辑页应有多个原生文本/形状/表格/图表对象。
5. 关键术语能作为原生文本检索到。
6. 中文不乱码、不重叠、不溢出、不小到不可读。
7. 已导出预览图和联系表。
8. 报告输出路径、页数、任务模式、审批门是否使用/跳过、可编辑性验证方式和仍需人工确认的点。
```
